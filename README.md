# Avito Seller Dashboard

Веб-приложение для личного кабинета продавца с интегрированным AI-ассистентом.  
Продавец может просматривать список своих объявлений, открывать карточку товара, редактировать данные и получать AI-рекомендации для описания и рыночной цены.

## Что реализовано

- страница списка объявлений `/ads`
- страница просмотра объявления `/ads/:id`
- страница редактирования объявления `/ads/:id/edit`
- поиск, сортировка, фильтрация и пагинация
- переключение layout списка: сетка / список
- вычисление признака `Требует доработок`
- сохранение черновика формы в `localStorage`
- AI-запросы для описания и цены через Ollama
- уведомления об успешном сохранении и ошибках
- 404 страница для неверных маршрутов

## Запуск проекта (интеграция с Ollama)

## Запуск с Docker

- Для скорости и точности модели лучше разворачивать приложение без Docker, см. раздел [Локальный запуск](#локальный-запуск-без-docker).

### Development сборка
```bash
docker-compose up --build
```

### Production сборка
```bash
docker-compose -f docker-compose.prod.yml up --build
```

Примечание:
- При первом запуске Docker скачает образ `ollama/ollama` (несколько GB).
- При первом запуске будет автоматически скачана лёгкая модель `llama3.2:3b` в Docker volume `ollama_data`.
- Запросы могут идти медленно из-за ограничений RAM в контейнере (можно увеличить в настройках Docker).

Дополнительно:
- Если нужно качество выше, можно вручную поставить `llama3`:
  - В файле `docker-compose.yml` изменить значение entrypoint на `ollama serve & sleep 2; ollama show llama3 >/dev/null 2>&1 || ollama pull llama3; wait`

Очистка места от Ollama:
```bash
docker-compose down
docker-compose -f docker-compose.prod.yml down
docker volume rm avito-test_ollama_data
```

## AI-сценарии

### Узнать рыночную цену

AI анализирует:
- категорию
- название
- описание
- характеристики товара

В ответе формируется:
- несколько ценовых диапазонов по состоянию
- средняя рекомендованная цена

При нажатии `Применить` средняя цена подставляется в поле формы.

### Придумать / улучшить описание

AI анализирует карточку объявления и предлагает более короткое и информативное описание.  
Если поле описания пустое, интерфейс предлагает `Придумать описание`, если заполнено — `Улучшить описание`.

## Принятые решения

### 1. Серверное состояние хранится через TanStack Query

Для списка, деталей объявления и сохранения изменений использован `TanStack Query`.  
Это дало:

- кэширование данных
- инвалидацию после сохранения
- optimistic update при редактировании
- поддержку отмены GET-запросов через `AbortSignal`

### 2. Черновики формы сохраняются в localStorage

Форма редактирования сохраняет промежуточное состояние в `localStorage`, чтобы пользователь мог восстановить данные после случайного обновления страницы.

### 3. Минимальные изменения backend

Для корректной работы frontend были внесены минимальные изменения в backend:

- в `GET /items` добавлен `id`
- в `GET /items` добавлена сортировка по `price`
- в `PUT /items/:id` разрешено изменение `category`
- добавлена базовая CORS-поддержка для `PUT/OPTIONS`

Изменения сведены к минимуму, без перестройки архитектуры сервера.

### 4. Поведение backend при сохранении

Backend использует in-memory хранение данных поверх `items.json`.  
Это означает:

- изменения доступны сразу после сохранения
- изменения сохраняются, пока жив процесс сервера
- после перезапуска backend данные возвращаются к исходному состоянию из JSON

### 5. Вычисление признака «Требует доработок»

Признак доработки опирается на:

- наличие `description`
- заполненность параметров категории

На странице просмотра также показывается список конкретных незаполненных полей.

### 6. Работа с изображениями

Реальные фотографии в задании не требовались, поэтому интерфейс использует placeholder-изображения.  
Для галереи на странице просмотра предусмотрена структура, которая позволяет в дальнейшем подставить реальные изображения без переработки layout.

## API

- `GET /items/:id` — получить объявление по ID
- `PUT /items/:id` — обновить объявление
